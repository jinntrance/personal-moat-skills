# Monthly Nine-Grid Review

## 概览

使用此技能执行月度个人运营复盘：先收集证据，再以九宫格为主线检查工作与生活系统，校准年度目标是否真正落到月计划、周计划和日行动，清理承诺和物品，综合个人笔记，最后设计下月行动。

保持证据化。某个数据源不可用时，不要阻塞流程，用已有证据继续，并在报告中明确标记缺口。

## 方法锚点

借鉴《只管去做》时，只保留可执行的结构，不复述书籍内容：

- 用愿景和年度九宫格提供方向，月复盘负责检查方向有没有落地。
- 把目标分成结果、项目、习惯三类：结果看进度，项目看里程碑，习惯看连续性。
- 用 SMART 检查下月任务是否具体、可衡量、能做到、有关联、有时间边界。
- 用项目计划倒推关键动作，再落到第一周和每天可做的 Next Action。
- 用 4D 思路处理任务：立刻做、计划做、授权/委托、删除。
- 反向检查计划失败的常见问题：不是真目标、过大、空泛、过多。下月计划要尽量真、小、实、少。

## 时间范围

- 如果用户指定月份，使用该月份。
- 如果用户未指定月份，且当前日期在每月前 7 天，默认复盘上一个完整自然月。
- 如果用户未指定月份，且当前日期在每月第 8 天或之后，默认复盘当月至今，并明确标注为「月中复盘」。
- 收集证据前，先计算并写出精确起止日期。

## 数据源地图

收集或请用户提供最小可用证据集：

- 滴答清单/Dida：月度 Summary、Statistics、OKR 列表或时间线、已完成任务、逾期任务、P1/P2、Waiting、Someday/Maybe、习惯打卡、已有日程任务。
- RescueTime：月度 Dashboard、分类时间分布、productive/distracting time、Top apps/sites、Highlights、未分类时间日志。
- 个人笔记：ima copilot 笔记、工作笔记、会议笔记、生活记录、阅读笔记、灵感、行动项。
- 可选补充：日历、iPhone Screen Time、博客统计、财务记录、过去几周的周复盘。

不要修改外部工具，除非用户明确要求。需要调整滴答清单时，先输出可审核的任务草案。

## 工作流

### 1. 建立证据账本

先创建一张可用证据表：

| 数据源 | 时间范围 | 状态 | 关键信号 | 缺口 |
|---|---|---|---|---|
| TickTick | YYYY-MM | 可用/缺失 | 完成、逾期、KR 进度 | - |
| RescueTime | YYYY-MM | 可用/缺失 | 高价值时间、分心时间 | - |
| ima notes | YYYY-MM | 可用/缺失 | 主题、决策、想法 | - |

用这张表防止模糊记忆压过真实记录。

### 2. 建立九宫格

优先使用用户已有九宫格分类；如果用户没有提供，使用默认九宫格：

| 格子 | 关注点 | 主要证据 |
|---|---|---|
| 工作/事业 | 关键产出、项目推进、职业杠杆 | TickTick、OKR、工作笔记、日历 |
| 个人项目/OKR | 自主目标、长期项目、关键结果 | OKR、任务完成、周复盘 |
| 学习成长 | 能力建设、课程、阅读、输出 | ima 笔记、读书笔记、Anki、博客 |
| 健康精力 | 睡眠、运动、工作时长、恢复 | 习惯打卡、Screen Time、健康日志 |
| 家庭/亲密关系 | 陪伴、沟通、承诺兑现 | 日历、任务、个人笔记 |
| 社交/合作 | 朋友、社群、协作、贵人网络 | 日历、会议笔记、消息转任务 |
| 财务/资产 | 收支、投资、风险、预算 | 财务记录、任务、笔记 |
| 环境/物品/系统 | 物品极简、数字整理、自动化 | 文件、截图、订阅、Someday |
| 心智/情绪/乐趣 | 情绪模式、压力源、意义感、休闲 | 个人笔记、时间日志、习惯 |

每个格子都输出：本月事实、评分、正常/超预期/不符预期、根因、保留动作、停止动作、下月推进项。

九宫格必须和年度方向相连：

- 如果用户有年度九宫格或年度 OKR，先把本月九宫格映射回年度格子。
- 如果没有年度九宫格，用本月九宫格反推当前真实重心，并标记「临时九宫格」。
- 每个格子最多保留 1 个下月主推进项；没有必要推进的格子明确写「维护」或「不动作」。

### 3. 复盘 OKR 与任务系统

按 KR 粒度检查月度进展，并映射到对应九宫格：

- 每个 KR 记录计划进度、实际进度、证据、状态和根因。
- 状态标记为：超前、正常、滞后、暂停、已不相关。
- 区分「结果问题」和「系统问题」：KR 不清晰、时间投入不足、外部依赖阻塞、精力或健康限制、优先级漂移。
- 对逾期、Waiting、Someday/Maybe 做月度清理：删除、降级、委托、转入下月、或放入长期停车场。
- 用 4D 处理任务池：2 分钟内能完成的立刻做；需要推进的计划到日历或任务系统；可由他人推进的委托；不再重要的删除。

### 4. 分析时间与注意力

把 RescueTime 和 Screen Time 当作注意力预算来复盘：

- 对比时间流向与 OKR 价值来源。
- 找出带来多数价值的 20% 活动，以及最低 ROI 的注意力消耗。
- 检查重复分心、未分类日志、通知、群、订阅源和应用。
- 将结论转化为：删除、屏蔽、批处理、迁移到电脑、自动化、委托、或有意识保留。
- 将时间结论回填到九宫格：哪些格子投入过少，哪些格子投入很多但产出很低。

### 5. 挖掘个人笔记

把 ima copilot 和个人笔记作为第二记忆：

- 提取决策、承诺、反复出现的担心、未闭环事项、想法和有用片段。
- 将承诺转成滴答清单风格的任务草案：动词、对象、截止时间、负责人、关联 KR。
- 只有当洞见会改变未来检索、决策或行为时，才提升为知识笔记或 Anki。
- 有趣但本月不可行动的想法，归入 Someday/Maybe。
- 阅读笔记只作为「学习成长」格子的证据；除非用户明确要求，不单独展开成独立专题。

### 6. 极简清理

生成两个明确清理清单：

- 不再做的事：任务、目标、会议、群、订阅、自动化、信息源、习惯、项目。处理方式为删除、延后、委托或降级。
- 不再保留的物品：实体物品、文件、截图、标签页、笔记和数字杂物。

核心问题：「如果它从下个月消失，会造成什么实质损失？」

### 7. 规划下月

把复盘转成设计：

- 每个活跃 KR 至少有一个月度推进项和一个第一周 Next Action。
- P1/P2 任务必须通过 SMART 检查。
- 应用 2/8 原则：识别少数最可能创造主要价值的行动。
- 设置 120% 挑战难度前，先检查时间容量和健康约束；不要让九个格子同时 120%。
- 更新已有日程，并为深度工作、行政事务、关系、运动和恢复保留时间。
- 提前一个月预判风险：Deadline 堆叠、外部依赖、出行、健康、资金、注意力、情绪负载。
- 下月计划必须落到三层：月度重点、第一周行动、每日可执行的最小动作。

## 质量标准

- 下月行动优先控制在 5-9 个关键动作，而不是生成很长的愿望清单。
- 九宫格必须是主输出；每个格子至少有一个证据点、一个判断和一个下月动作或明确不动作。
- 必须体现「年度方向 -> 月度重点 -> 第一周行动 -> 每日最小动作」的落地链。
- 下月任务必须通过真、小、实、少检查：真目标、可变小、可执行、不过载。
- 重要判断必须能指向滴答清单、RescueTime、笔记或用户明确记忆。
- 区分事实、解释和决策。
- 对不确定性保持标注，不用自信猜测填补数据缺口。
- 最终必须产出可立即执行的任务草案、日程调整建议和 not-to-do 决策。
